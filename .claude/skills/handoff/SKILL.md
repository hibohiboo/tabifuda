# Handoff

正は docs/agent-operations.md「コンテキスト管理とハンドオフ」。ここは実行手順のみ。

## 渡す側(60%に達したら新しい作業に着手しない)

1. WIPコミットを切る(新セッションが git diff とテスト結果から状態復元
   できるようにする)
2. `docs/handoff/<日付>-<トピック>.md` にメモを書く。項目は固定:
   - 目標 / 完了済み / 残作業 / 次の一手 / ハマりどころ / 関連ファイルパス
3. **仕様・決定事項をメモに書かない。** 書きたくなったら、それは
   docs/design/ 等に属する内容なので先にそちらへ移す(正を二重化しない)
4. 60%に頻繁に達するなら、タスクの切り方が大きすぎるシグナルとして
   人間に分割を提案する

## 受ける側(新セッション)

1. docs/handoff/ のメモ + CLAUDE.md + メモが指す設計文書を読んで再開する
2. `git diff` とテスト実行で実際の状態を確認する(メモより実物を信じる)
3. 引き継ぎ完了後、メモを削除する(使い捨て)

## 計測

自動圧縮(auto-compact)が発動していたら失敗として docs/agent-journal.md に
1行記録する(タスク設計の失敗として週次棚卸しで扱う)。
