# Decision Log

人間の判断待ちがセッションを跨ぐ論点を、git証跡付きで進める手順。
**詳細と変更は docs/agent-operations.md「人間の判断が要る論点の進め方」が正。**
文書側を変えたらこのスキルも同PRで更新する。

## いつ使うか

- レビュー指摘のうち、仕様判断を伴い人間の決定が要るものが複数残ったとき
- 設計上の選択肢を提示して人間の回答を待つ状態で、セッションが切れうるとき
- 「前回の判断の続きから」と言われたとき(再開側)

## 手順(新規作成)

1. `docs/tasks/plans/<対象>-decisions.md` を作る。構成:
   - 冒頭: 元文書への参照、運用ルール、**状態表**(項目×状態×決定内容)
   - 論点ごとの節: 論点 / 選択肢 / 決定(未定) / 反映先。
     **新セッションがこの文書だけで再開できる自己完結度**にする
2. 処理順は「早くブロックする順」(先のサイクルを塞ぐものから)
3. 作成したら即コミット(全項目「未着手」の状態で証跡開始)

## 手順(進行)

- 状態は 未着手 → 議論中 → 決定済み → 反映済み の4段階
- **状態が変わるたびに文書を更新してコミット**(議論中の中断も証跡に残す)
- 1論点ずつ: ①論点提示 → ②人間が決定 → ③設計文書・実装へ反映 →
  ④決定ログ更新。まとめて聞かない
- 決定時は決定内容だけでなく**理由**も記録する

## 手順(再開側)

1. 対象の decisions.md の状態表を見る
2. 最初の未完了項目から続きを開始(議論中なら、記録済みの経緯を要約して
   人間に「ここまで話した」を提示してから再開)

## 規律

- **正を二重化しない**: 決定した仕様は docs/design/ 等へ反映するのが正。
  決定ログは経緯と証跡(handoffと同じ規律)
- 全論点が反映済みになったらクローズ。削除はしない(意思決定の由来を残す)
