# Phase 3.5 実装タスク: CLI永続化(中断・再開/パーティ/持ち帰り)

実行モデル: Sonnet 5。1サイクル=1セッション=1PR。
開始前の儀式は phase2-task.md 冒頭と同じ(文書確認→差分報告→改訂→着手)。

**番号について**: P3(Web)とP4(バックエンド)の間に後から挟んだフェーズ
(2026-07-20)。既存のADR・決定ログが「P4=バックエンド」の意味で多数参照して
いるため、番号は振り直さず P3.5 とした。依存は crates/ のみで P3 とは独立して
おり、**P3 より先に着手してもよい**。

## 目的

セッションを中断しても再開でき、パーティがセッションを跨いで持続し、
クリア時に持ち出し可(portable)カードを持ち帰れる状態にする。
将来要望メモ §1(中断・再開の部分)と §3(報酬とセッション終了処理)の採用。

## 人間の事前決定(該当サイクルの前に確認)

- **Event追加の承認**(C3): `RewardsGranted` / `CardsDiscarded`
  (CLAUDE.md「Event / Command / PatchOp の追加は要相談」)
- **portable の表現**(C3): `CardDef.portable: bool` か、予約済み `tags` の
  `#portable` か(将来要望メモ §4 タグシステムとの整合を比較提示して選ぶ)
- **セーブデータの互換性方針**(C1): フォーマットversionを埋めるか。
  Event enum は `#[non_exhaustive]` で追加前提のため、古い保存ファイルの
  読み込み方針(拒否/警告付き読込)を決める

## 着手時の文書運用

§1・§3 の該当部分を採用するサイクルでは、着手時に future-requirements.md の
該当記述を規範文書(domain-model.md 等)へ移し、メモから削除する
(future-requirements.md 冒頭の運用ルール。§3 には 2026-07-20 決定済みの
方針——既定不可・明示で可、CardDef凍結コピー、シナリオ非依存制約——が
記録されている)。

## サイクル

### C1: セッションの保存と再開

- 保存形式の設計を先に文書化(イベント列のシリアライズ。`SessionStarted` に
  シナリオ・パーティの凍結コピーが入るためファイルは自己完結。
  リプレイ決定性は P1 の replay_tests で固定済み)。置き場は P3 C0 の
  レイヤ別文書の決定に従う(未決なら domain-model.md にCLIの決定として)
- `q` 中断時に保存を尋ね、`play --resume <session-file>` でイベント列を
  apply で畳み込んで復元し、続きから遊べる
- 保存→再開→最後まで、の結合テスト(**Paused中の中断→再開で裁定待ちに
  戻る経路を含める**)
- セーブファイルは自由入力本文(冒険記=ドメインログ)を含む。
  運用ログに本文を書かない規律(cross-cutting.md)は従来どおり

### C2: パーティのload/save

- パーティファイル形式(`Vec<Character>` のJSON)を設計文書化
- `play --party <party-file>`。無指定は従来のソロ既定(「旅人」)を維持
- 終了時の書き戻しは C4 の finalize 配線が担うため、C2 は読み込みと
  ファイル検証まで

### C3: 報酬と持ち帰り(コア)

- §3 の採用: 持ち出しは**既定不可・明示のみ可**、`Marker` は常に不可、
  持ち帰りは CardDef の**凍結コピー**(`Character.owned_cards` 等。
  CardId 参照はシナリオ外で解決不能なため)
- finalize を decide/apply に実装(Event追加は事前決定を経る)。
  正常系+拒否系(非portableは持ち出されない)を対で書く
- lint拡張: portable カードの effects/requires がシナリオ非依存であること
  (`GotoScene` / `AdvancePhase` / シナリオ固有CardIdへの参照を含むカードは
  portable にできない)

### C4: 持ち帰りのCLI配線

- セッション終了時、portable カードの持ち帰り→パーティファイルへの書き戻し
- demo.md 更新(中断・再開・パーティ指定・持ち帰りの操作)
- 複数セッションファイルによる並行プレイが自然に成立することの確認
  (§1 の「同一キャラの並行参加制約」は将来要望のまま残す)

## 完了条件

中断→再開で通しプレイが最後まで可能(Paused中の中断を含む) /
パーティがセッションを跨いで持続し、portable カードの持ち帰りがテストで固定 /
非portable カードが持ち出されない拒否系テストがある /
lint拡張(シナリオ非依存検査)が入っている
