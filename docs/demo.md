# 通しプレイ手順(デモ用・非規範)

**読者**: 企画・PO。実際に手元で「単純討伐」を遊んでみるための手順書。
**位置づけ**: 操作ガイドであり仕様ではない。ゲームの世界観・概念は
[design/domain-guide.md](design/domain-guide.md) を先に読むと理解しやすい。
規範(正)は [design/domain-model.md](design/domain-model.md)。

---

## 1. 準備するもの

- Rust(rustup)。https://rustup.rs から導入する。バージョンは
  `rust-toolchain.toml` に固定されており、リポジトリ内で `cargo` を
  実行すると自動的に合わせてくれる
- ターミナル(Windows/Mac/Linuxいずれでも可)
- このリポジトリを取得済みであること

## 2. 起動する

リポジトリのルート(`tabifuda/`)で、次のコマンドを実行する。

```
cargo run -p tabifuda-cli -- play shared/scenarios/simple-hunt.json
```

初回はビルドのため数十秒かかる。2回目以降は数秒で起動する。

## 3. 画面の読み方・操作方法

各ターン、次の情報が表示される。

- **シーンの描写**(いまどこで何が起きているか)
- **手札**: `[1] カード名` のような番号つきリスト
- **コマンド**: `番号=カードを出す / p=提案する / q=中断`

操作は次の3種類。

| 入力 | 意味 |
|---|---|
| 数字(例: `1`) | その番号のカードを出す |
| `p` | 「提案」を出す(台本にない行動をGMに相談する) |
| `q` | 途中で中断する(いつでも可) |

**台詞カード**(「〜を受ける」「〜を告げる」等)を出すと、追加で
「自由入力(任意。Enterでスキップ)」と聞かれる。セリフを打ち込んでも、
何も入力せず Enter だけでスキップしてもよい(スキップしてもカードは
出せる)。

**提案(`p`)を出すと、次のターンで「あなたがGMとして」採決を求められる**
(ソロプレイのため、1人がプレイヤーとGMの両方を兼ねる。
domain-guide.md「GMの介入」参照)。`y`(採用)/`n`(却下)で答える。

> 現時点の仕様として、出したカードは手札から消えない(そのまま
> 残り続ける)。同じカードが選択肢に残っていても不具合ではない。

## 4. 討伐に成功するルート(勝利エンド)の操作例

上から順に入力していくと、依頼を受ける→森へ向かう→獣と戦う→勝利する、
という一本道を最後まで体験できる(途中で提案→GM裁定も1回試す)。

| 画面の状態 | 入力する内容 |
|---|---|
| オープニング。手札に `[1] 依頼を受ける` | `1` と入力 |
| 「自由入力(任意。Enterでスキップ)」 | 何も入力せず Enter |
| ミドル。手札に `[3] 獣の巣に到着する` が増えている | `3` と入力 |
| クライマックス。手札に `[4] 打ち倒す` `[5] 退く` が増えている | `p` と入力(提案を試す) |
| 「提案内容を入力してください」 | 好きな文章(例: `近道を探したい`) |
| 「採用しますか? [y/n]」 | `y` と入力 |
| クライマックスに戻る | `4`(打ち倒す)と入力 |
| エピローグ。手札に `[6] 村に帰還を告げる` が増えている | `6` と入力 |
| 「自由入力(任意。Enterでスキップ)」 | 好きな一言、または Enter でスキップ |

最後に `=== 冒険の終わり: Victory ===` と表示され、続けてここまでの
出来事をまとめた**冒険記**(あらすじ全文。提案の内容やセリフも含む)が
表示されて終了する。

## 5. 失敗するルート(敗北エンド)を見たい場合

上記と同じ手順で、クライマックスの手札選択のときだけ `[5] 退く` を
選ぶと、`epilogue_lose` シーンに分岐し、`=== 冒険の終わり: Defeat ===`
で終わる。

## 6. うまく動かないとき

| 症状 | 対処 |
|---|---|
| `cargo: command not found` | Rustが未導入。https://rustup.rs から導入する |
| 文字が崩れる | ターミナルの文字コードをUTF-8にする |
| 途中でおかしくなった | `q` で中断し、`cargo run -p tabifuda-cli -- play ...` を撃ち直せば最初からやり直せる(セッションは保存されない) |

## 7. シナリオデータを覗いてみたい場合

「単純討伐」のシナリオデータ本体は `shared/scenarios/simple-hunt.json`。
テキストエディタで開けば、どのシーンでどのカードが配られ、どんな効果を
持つかがそのまま読める。データの整合性チェックは次のコマンドで行える
(壊れていなければ `ok: no issues found` と表示される)。

```
cargo run -p tabifuda-cli -- lint shared/scenarios/simple-hunt.json
```
