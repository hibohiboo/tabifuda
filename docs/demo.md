# 通しプレイ手順(デモ用・非規範)

**読者**: 企画・PO。実際に手元で「単純討伐」を遊んでみるための手順書。
**位置づけ**: 操作ガイドであり仕様ではない。ゲームの世界観・概念は
[design/domain-guide.md](design/domain-guide.md) を先に読むと理解しやすい。
規範(正)は [design/domain-model.md](design/domain-model.md)。

---

## 1. 準備するもの

- Rust(rustup)。https://rustup.rs から導入する。バージョンは
  `rust-toolchain.toml` に固定されており、リポジトリ内で `cargo` を
  実行すると自動的に合わせてくれる
- ターミナル(Windows/Mac/Linuxいずれでも可)
- このリポジトリを取得済みであること

## 2. 起動する

リポジトリのルート(`tabifuda/`)で、次のコマンドを実行する。

```
cargo run -p tabifuda-cli -- play shared/scenarios/simple-hunt.json
```

初回はビルドのため数十秒かかる。2回目以降は数秒で起動する。

## 3. 画面の読み方・操作方法

各ターン、次の情報が表示される。

- **シーンの描写**(いまどこで何が起きているか)
- **手札**: `[1] カード名` のような番号つきリスト
- **コマンド**: `番号=カードを出す / p=提案する / q=中断`

操作は次の3種類。

| 入力 | 意味 |
|---|---|
| 数字(例: `1`) | その番号のカードを出す |
| `p` | 「提案」を出す(台本にない行動をGMに相談する) |
| `q` | 途中で中断する(いつでも可) |

**台詞カード**(「〜を受ける」「〜を告げる」等)を出すと、追加で
「自由入力(任意。Enterでスキップ)」と聞かれる。セリフを打ち込んでも、
何も入力せず Enter だけでスキップしてもよい(スキップしてもカードは
出せる)。

**提案(`p`)を出すと、次のターンで「あなたがGMとして」応答を求められる**
(ソロプレイのため、1人がプレイヤーとGMの両方を兼ねる。
domain-guide.md「GMの介入」参照)。GMの応答は3種類。

| 入力 | 意味 |
|---|---|
| `y` | 提案を採用して再開する |
| `n` | 提案を却下して再開する(却下の記録も冒険記に残る) |
| `c` | **カードを配って応える**。カード名と回答文を入力すると、その場で手札に質問カードが追加される。プレイヤーがそれを出すと回答文が表示される。配った後も裁定待ちのままなので、複数枚配ってから `y`/`n` で締める |

> 出したカードは手札から消える。また、選ばなかった側の選択肢カード
> (例: クライマックスで「打ち倒す」を選ぶと「退く」)も、そのシーンを
> 離れると自動的に消える。世界の状態を示す「印」(例: 依頼を受けた
> ことを示す「依頼受諾」)は選ぶ対象ではないため、そもそも一覧に出ない。
> **このため手札の番号は毎回`[1]`から振り直される**(下表の番号は
> すべて`[1]`基準)。

## 4. 討伐に成功するルート(勝利エンド)の操作例

上から順に入力していくと、依頼主に質問する→依頼を受ける→森へ向かう→
獣と戦う→勝利する、という流れを最後まで体験できる(途中で
提案→GMがカードを配って応える、も1回試す)。

| 画面の状態 | 入力する内容 |
|---|---|
| オープニング。手札に `[1] 依頼を受ける` | `p` と入力(提案を試す) |
| 「提案内容を入力してください」 | `獣の姿や被害を知りたい` |
| 「提案が届いています」`[y/n/c]` | `c` と入力(カードを配って応える) |
| 「カード名」 | `獣の目撃情報を尋ねる` |
| 「回答文(カードを出すと表示)」 | `銀色の毛並みの大狼だという。家畜が三頭襲われた。` |
| `[y/n/c]` に戻る | `y` と入力(採用して再開) |
| オープニング。手札に `[1] 依頼を受ける` `[2] 獣の目撃情報を尋ねる` | `2` と入力 |
| 回答文が表示され、オープニングに戻る | (読んだら)`1` と入力 |
| 「自由入力(任意。Enterでスキップ)」 | 何も入力せず Enter |
| ミドル。手札に `[1] 獣の巣に到着する` | `1` と入力 |
| クライマックス。手札に `[1] 打ち倒す` `[2] 退く` | `1`(打ち倒す)と入力 |
| エピローグ。手札に `[1] 村に帰還を告げる` (「退く」は自動的に消えている) | `1` と入力 |
| 「自由入力(任意。Enterでスキップ)」 | 好きな一言、または Enter でスキップ |
| 「フォークとして保存しますか? [y/n]」 | `n`(保存を試すなら `y`。下記参照) |

最後に `=== 冒険の終わり: Victory ===` と表示され、続けてここまでの
出来事をまとめた**冒険記**(あらすじ全文。提案の内容やセリフも含む)が
表示されて終了する。

セッション中にカードを配って応えるなどの**改編(パッチ)**をしていた場合、
終了時に「フォークとして保存しますか?」と聞かれる。`y` と入力すると、
改編を取り込んだ**別シナリオ**が元ファイルの隣(例:
`shared/scenarios/simple-hunt-fork.json`)に保存される。元のファイルは
変更されない。保存したフォークはそのまま
`cargo run -p tabifuda-cli -- play shared/scenarios/simple-hunt-fork.json`
で遊べ、セッション中に配ったカード(例: 獣の目撃情報を尋ねる)は
**同じ場面で最初から配られる**(次のセッションでも改編が活きる)。

## 5. 失敗するルート(敗北エンド)を見たい場合

上記と同じ手順で、クライマックスの手札選択のときだけ `[2] 退く` を
選ぶと、`epilogue_lose` シーンに分岐し、`=== 冒険の終わり: Defeat ===`
で終わる。

## 6. うまく動かないとき

| 症状 | 対処 |
|---|---|
| `cargo: command not found` | Rustが未導入。https://rustup.rs から導入する |
| 文字が崩れる | ターミナルの文字コードをUTF-8にする |
| 途中でおかしくなった | `q` で中断し、`cargo run -p tabifuda-cli -- play ...` を撃ち直せば最初からやり直せる(セッションは保存されない) |

## 7. シナリオデータを覗いてみたい場合

「単純討伐」のシナリオデータ本体は `shared/scenarios/simple-hunt.json`。
テキストエディタで開けば、どのシーンでどのカードが配られ、どんな効果を
持つかがそのまま読める。データの整合性チェックは次のコマンドで行える
(壊れていなければ `ok: no issues found` と表示される)。

```
cargo run -p tabifuda-cli -- lint shared/scenarios/simple-hunt.json
```
